# 日本編 2016〜2017年：ピゴサ・マジゴー・日本固有エコシステム

## 2016年：日本リリースと「聖地」文化

### 日本リリース（2016年7月22日）

海外より約2週間遅れて日本でリリース。「聖地」文化（世田谷公園・天保山等）が生まれ、希少ポケモンの出現スポット情報がSNSで拡散。この聖地文化がスキャナー需要を生む土壌となった。

### P-GO SEARCH（ピゴサ）誕生

開発者「くくさま」（@kukusama、kuku.lu）による個人開発のスキャナー。日本リリース直後から開発・公開された。

**主な機能：**
- 地図上でポケモンの出現位置をリアルタイム表示
- 個体値・消滅時刻・レイド情報表示
- プッシュ通知対応
- iOS/Android/Web対応

PokeVision閉鎖後も生き残った日本唯一の大規模スキャナー。「ピゴサをやるためにポケモンGOをやっているようなもの」という状態のユーザーが多数存在した。

### スプーフィング（2016年当時）

- **Android**：開発者オプション「モック位置情報」＋GPS Joystickで比較的容易
- **iOS**：TutuApp＋PokeGo++の組み合わせ。JB不要で動作

### 海外スプーファーが「日本座標」を狙う

ガルーラ（日本・アジア限定地域ポケモン）目的の海外スプーファーが日本のGPSへテレポートする問題が話題に。日本がスプーフィングの「目的地」になるという逆輸入的な現象が発生。

## 2017年：ピゴサ閉鎖とマジゴー登場

### コイルアップデート（2016年10月）でピゴサ一時停止

API v0.39でピゴサが一時停止。しかし11月に復活し、引き続き日本最大のスキャナーとして機能した。

### ピゴサ閉鎖（2017年7月23日）

1周年前日にTPC（The Pokémon Company International）からの要請を受けて閉鎖。くくさまが公式FAQを発表し「利用者へのペナルティはない」と明言。2chやTwitterに感謝コメントが殺到した。

:::message
ピゴサの閉鎖は日本のポケモンGOコミュニティにとって一つの時代の終わりとして記憶されている。
:::

### マジカルGO（マジゴー）登場

ピゴサ後継として登場したスキャナー。製作者不明のまま運営が開始された。ピゴサとほぼ同一UIを持ちiOS/Android/Web対応。レイドボス情報・個体値・通知機能も継承。

### TutuApp＋PokeGo++が日本でも普及

日本の攻略サイトでTutuApp経由のPokeGo++インストール手順が公開され始める。「JB不要で使える」という手軽さから一般ユーザー層にも広まった。

### GOフェスト シカゴ失敗の日本語報道

2017年7月22日のGOフェスト シカゴ大失敗は日本でも大きく報道。「ナイアンの杜撰な運営」として日本のゲームメディアが批判的に報道した。

### レイドバトル実装と日本コミュニティ

2017年6月29日のレイドバトル実装後、日本でも伝説レイド参加目的のスプーフィングが増加。Discord上で「〇〇のジムにメタグロスが出た」という座標共有が始まる。

## 日本固有の特徴（2016〜2017年）

| 特徴 | 内容 |
|---|---|
| 聖地文化 | 特定スポットに大量出現→スキャナー需要を加速 |
| ピゴサ | 日本製・日本語対応スキャナーの独自発展 |
| ガルーラ問題 | 日本限定ポケモン目当ての海外スプーファー |
| 2ch文化 | 5chへ移行しつつ、ポケGO情報の中心地に |
